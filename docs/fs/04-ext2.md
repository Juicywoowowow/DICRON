EXT2 Filesystem
===============

1. Overview
-----------

EXT2 (Second Extended Filesystem) is a disk-based filesystem for
persistent storage. DICRON includes an EXT2 implementation in
src/drivers/ext2/.

2. Structure
------------

EXT2 divides disk into:
  - Superblock: Filesystem metadata
  - Block groups: Group of blocks with own inode/bitmap
  - Inodes: File metadata structures
  - Data blocks: Actual file content

3. Superblock
-------------

The superblock contains:
  - Inodes count
  - Blocks count
  - Block size (log2)
  - Blocks per group
  - Inodes per group
  - Magic number (0xEF53)
  - Revision level

4. Inodes
---------

Each inode stores:
  - Mode (file type, permissions)
  - Size
  - Timestamps (access, modify, change)
  - Direct block pointers (12)
  - Indirect block pointers
  - Double indirect pointers
  - Triple indirect pointers

5. Features Implemented
-----------------------

  - Regular files and directories
  - Create, read, write, unlink
  - mkdir and rmdir
  - Block allocation and freeing
  - Bitmap-based allocation
  - Symbolic links (optional)

6. API
------

  void ext2_init(void)
      - Initialize EXT2 driver

The driver provides VFS operations through the inode_operations
and file_operations interfaces.

7. Implementation Files
-----------------------

  ext2.h           - Main header
  ext2_super.c     - Superblock handling
  ext2_bitmap.c    - Block/inode bitmap management
  ext2_inode.c     - Inode operations
  ext2_dir.c       - Directory operations
  ext2_file.c      - File operations
  ext2_block.c     - Block allocation
  ext2_write.c     - Write operations
  ext2_ops.c       - VFS operations
  ext2_vfs.c       - VFS registration

8. Block Allocation
------------------

EXT2 uses bitmaps to track free blocks and inodes:
  - Block bitmap: 1 bit per data block
  - Inode bitmap: 1 bit per inode

9. Limitations
-------------

  - No journaling (can corrupt on power failure)
  - No extended attributes
  - No ACLs
  - Fixed inode count at mkfs time

10. Disk Layout
--------------

  [Boot Sector] [Superblock] [Group Descriptors] [Block Bitmap] [Inode Bitmap] [Inode Table] [Data Blocks]
